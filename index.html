Faculty Presentation

Project Overview — For Faculty

Complete technical summary of concepts, stack, and outcomes.

🎯
Problem Statement

EV drivers in India face uncertainty about charging station availability, leading to range anxiety and poor planning. This system uses Machine Learning to predict availability and recommend the optimal station in real-time based on the user's GPS location.

🤖
ML Models Used
Random Forest Regressor — Predicts availability_percentage (continuous output). Ensemble of 100 decision trees. MAE ≈ 3.2%.
Decision Tree Classifier — Predicts congestion level (Low/Medium/High). max_depth=6. Accuracy ≈ 93%.
Scoring Algorithm — Combines normalized distance + busy score to rank stations.
📦
Tech Stack
🐍 Python — Core language
🌐 Flask — REST API backend
🧠 scikit-learn — ML models
🐼 Pandas / NumPy — Data processing
🗺️ Leaflet.js — Interactive maps
📊 Plotly — Analytics dashboard
🎨 HTML5 / CSS3 / JS — Frontend
📊
Dataset
Training Data: 8,000 synthetic samples with features: hour, day, temperature, rain → target: availability %, congestion level
Stations Data: 130 stations across 56 cities and 19 states — manually curated with real-world GPS coordinates
Features: station name, city, state, lat/lon, charger type, total slots, busy level, wait time
🔁
System Flow
1️⃣ User provides GPS / selects city
2️⃣ User inputs hour, day, temp, rain
3️⃣ Flask sends features to RF + DT models
4️⃣ Models return availability % + congestion
5️⃣ Backend scores all stations by distance + busy
6️⃣ Top 10 results returned with wait estimates
7️⃣ Frontend renders map + directions + chart
8️⃣ Smart suggestions generated from ML output
✅
Key Outcomes
✅ Real-time GPS station discovery

✅ Smart charging time suggestions
✅ Estimated wait time per station
✅ Interactive map with route & directions
✅ Google Maps deep-link navigation
✅ All-India coverage: 56 cities, 19 states
✅ Responsive, production-grade UI
